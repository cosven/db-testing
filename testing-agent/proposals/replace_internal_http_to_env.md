Proposal：内网 http 地址转 env

背景
- 该仓库将 push 到公网，内网地址不应出现在源码中。
- 本次范围仅处理内网地址（大概率所有 http 都是内网）。

目标
- 将内网 http URL 统一替换为环境变量读取。
- 保持现有运行行为，提供 .env 默认值或明确报错提示。

范围
- 本仓库内的脚本、文档、配置文件。
- 仅替换内网 http URL（如 172.* 或私网域名）。

不做
- 不修改外网/可公开地址。
- 不删除文件或功能。

计划（逐步确认）
1) 盘点：列出所有 "http://" 出现位置并标注内网项。
2) 映射：设计 env 命名与默认值，更新 .env.example。
3) 替换：按文件逐一替换并回归检查。
4) 复核：再次搜索确认无内网 http 残留。

Env 命名建议
- 按服务命名：PROMETHEUS_URL、CLUSTER_API_URL、JENKINS_URL 等。
- .env.example 使用示例地址。

风险与应对
- 风险：遗漏隐藏在文档/脚本的内网地址。
  应对：全仓搜索 + 替换后复核。
- 风险：缺失 env 导致行为变化。
  应对：保留默认值或给出清晰报错。

验收
- 代码库内不再出现内网 http 地址。
- .env.example 已更新为示例地址。
